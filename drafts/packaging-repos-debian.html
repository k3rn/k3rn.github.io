Packaging & Repository

You may want to create a package for various reasons: to distribute your software publicly, to use a version that is not available on the repositories (for being newer or with a special set of features) or to deploy your application.

With pbuilder you can build your package in a insolate enviroment, ensuring that you specified the right build dependencies in debian/control. pbuilder also has hooks, that enables to automaticly run tests, such as one that ensures that the package's dependecies are satisfied and that it can be installed/removed/purged without causing erros and lintian that will dissects the package and try to find bugs and policy violations.

To create, build and sign the package you will need to install the following:

# apt-get install dh-make pbuilder gnupg rng-tools

Let's generate the GPG key:

$ sudo rngd -r /dev/urandom
$ gpg --gen-key

The first command will create a source of entropy, the second will generate the key. You can use the default options.
